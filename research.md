### Как презентовать AI-услуги на лендинге

1. **Живая демонстрация** — встроенный чат-бот или возможность «попробовать» агента
2. **Результаты в цифрах** — «сократили время с 4 часов до 30 секунд»
3. **Темная тема + неоновые/градиентные акценты** — стандарт AI-индустрии
4. **Показывать процесс** — схемы архитектуры, скриншоты workflow
5. **Узкая специализация** конвертирует лучше общего позиционирования
6. **Образовательный контент** — блог/видео, объясняющие AI простым языком

# Часть 2. СТРУКТУРА ЛУЧШИХ ЛЕНДИНГОВ

## 2.1 Универсальные блоки продающего лендинга

### Блок 1. Hero / Первый экран
- **Цель:** Захватить внимание за 3-5 секунд
- **Содержимое:** Заголовок-оффер, подзаголовок, CTA-кнопка, визуал (фото/видео)
- **Примеры:** Stripe.com, Linear.app, Tilda.cc

### Блок 2. Проблема / Боль клиента
- **Цель:** Создать эмоциональный резонанс — «это про меня»
- **Содержимое:** 3-4 типичные боли, визуализация, формулировки от первого лица
- **Примеры:** Basecamp, Monday.com

### Блок 3. Решение / Преимущества
- **Цель:** Показать, как продукт решает проблемы
- **Содержимое:** 3-6 преимуществ с иконками, акцент на результатах
- **Примеры:** Notion.so, Figma

### Блок 4. Как это работает / Процесс
- **Цель:** Снять неопределенность, показать простоту
- **Содержимое:** 3-5 пронумерованных шагов, визуальная линия
- **Примеры:** Dropbox, Tinkoff

### Блок 5. Портфолио / Кейсы
- **Цель:** Продемонстрировать экспертизу
- **Содержимое:** 3-6 проектов, скриншоты, задача -> результат, метрики

### Блок 6. Отзывы / Социальное доказательство
- **Цель:** Сформировать доверие
- **Содержимое:** 3-6 отзывов с фото, логотипы клиентов
- **Примеры:** Slack, Shopify

### Блок 7. Тарифы / Цены
- **Цель:** Прозрачность стоимости
- **Содержимое:** 2-4 тарифных плана, выделение «рекомендуемого»

### Блок 8. FAQ
- **Цель:** Снять оставшиеся возражения
- **Содержимое:** 5-8 вопросов в формате аккордеон

### Блок 9. Финальный CTA / Контакты
- **Цель:** Конвертировать «дозревших»
- **Содержимое:** Повторение оффера, форма, мессенджеры, карта

### Блок 10. Футер
- **Цель:** Навигация и юридическая информация

# Часть 3. ТРЕНДЫ ВЕБ-ДИЗАЙНА 2025-2026

## Тренд 1. Scroll-driven анимации (CSS native)
- **Описание:** Нативные CSS-анимации, привязанные к скроллу, без JavaScript
- **Технология:** `animation-timeline: scroll()` и `animation-timeline: view()`
- **Применение:** Появление секций при скролле, прогресс-бар, параллакс, печатающийся текст

## Тренд 2. Bento Grid Layout
- **Описание:** Модульная сетка в стиле Apple — карточки разного размера
- **Технология:** `display: grid` + `grid-template-areas`
- **Применение:** Секция «Преимущества» или «Функции», размеры 1x1, 2x1, 1x2, 2x2

## Тренд 3. Кинетическая типографика
- **Описание:** Текст как главный визуальный элемент. Крупные заголовки (5vw-15vw), анимированные буквы
- **Технология:** GSAP SplitText, Variable Fonts
- **Шрифты:** Satoshi, General Sans, Clash Display, Cabinet Grotesk, Space Grotesk

## Тренд 4. Mesh-градиенты и Oklch
- **Описание:** Многоцветные «жидкие» градиенты, приглушенные палитры
- **Технология:** CSS `oklch()`, многоточечные градиенты
- **Палитра 2025-2026:** Глубокий индиго, пыльная роза, шалфейный зеленый, электрический фиолетовый

## Тренд 5. Микровзаимодействия и Hover-эффекты
- **Описание:** Магнитные кнопки, 3D-наклон, cursor-following эффекты
- **Применение:** Tilt-карточки, кастомный курсор, анимированные underline

## Тренд 6. 3D-элементы и WebGL
- **Описание:** Интеграция 3D-объектов в веб-страницу
- **Технология:** Three.js, Spline, React Three Fiber
- **Применение:** 3D-модели продуктов, абстрактные шейдерные фоны, интерактивные частицы

## Тренд 7. Горизонтальный скролл
- **Описание:** Секции с горизонтальной прокруткой при вертикальном скролле
- **Технология:** GSAP ScrollTrigger `pin`
- **Применение:** Портфолио/кейсы, диагональные разделители через `clip-path`

## Тренд 8. Темный режим как основной
- **Описание:** Dark mode — основной режим, а не дополнение
- **Применение:** Фон #0a0a0a-#1a1a2e, неоновые акценты, glow-эффекты, смена тем между секциями

## Тренд 9. Lottie-анимации и SVG Motion
- **Описание:** JSON-анимации для легких иллюстраций, SVG line-анимации
- **Технология:** DotLottie, @lottiefiles/dotlottie-web
- **Применение:** Анимированные иконки, hero-иллюстрации, SVG-логотип с прорисовкой

## Тренд 10. Glassmorphism 2.0
- **Описание:** Тонкое размытие, слоистость, нойз-текстуры, зернистость
- **Технология:** `backdrop-filter: blur()`, SVG `feTurbulence`
- **Применение:** Навигация и карточки с glassmorphism, текстурные фоны

## Тренд 11. AI-персонализация на лендинге
- **Описание:** Чат-боты, многошаговые формы, динамический контент
- **Применение:** AI-чат-виджет, калькулятор стоимости, персонализация по UTM-меткам

```javascript
function animateCounter(el, target, duration = 2000) {
  const start = performance.now();
  function update(now) {
    const progress = Math.min((now - start) / duration, 1);
    const eased = 1 - Math.pow(1 - progress, 4);
    el.textContent = Math.floor(eased * target).toLocaleString();
    if (progress < 1) requestAnimationFrame(update);
    else el.textContent = target.toLocaleString();
  }
  requestAnimationFrame(update);
}
// Используется с IntersectionObserver — запуск при появлении на экране
```

# Часть 7. ЭКСПЕРТНАЯ ОЦЕНКА

## 7.1 UX-дизайнер (10 лет опыта)

### Ошибки, которые делают новички

**1. Две темы — ноль фокуса.** Человек заходит и за 3 секунды должен понять: «Это про что?» Когда одновременно продается разработка и недвижимость — мозг буксует, пользователь уходит.
> Решение: Четкая сегментация. Либо два раздела (`/dev` и `/phuket`), либо hero с явным выбором: «Я ищу разработчика» / «Я ищу недвижимость».

**2. Hero без конкретного оффера.** «Привет, я делаю крутые вещи» — это ни о чем. Нет ценностного предложения, нет причины остаться.
> Решение: Заголовок = конкретная выгода. Подзаголовок уточняет, кнопка — следующий шаг.

**3. Стена текста вместо структуры.** Пользователь сканирует, а не читает. Если за 5 секунд скроллинга не зацепился — ушел.
> Решение: Один экран — одна мысль. Заголовки, буллеты, воздух между секциями.

**4. Нет социального доказательства.** Голословные заявления без доказательств.
> Решение: Кейсы с результатами, отзывы с именами, логотипы клиентов, количество проектов.

**5. Один CTA на весь сайт.** Кнопка «Связаться» внизу — 90% до нее не доскроллят.
> Решение: CTA после каждого логического блока. Формулировки разные: «Обсудить проект», «Посмотреть объекты», «Получить консультацию».

**6. Портфолио как галерея скриншотов.** Десять картинок без контекста.
> Решение: Каждый кейс = задача -> решение -> результат.

**7. Игнорирование мобильной версии.** 60-70% трафика — мобильные.
> Решение: Mobile-first. Тап-зоны 44x44px. Тестирование на реальных устройствах.

**8. Перегруз анимациями.** Все летает, крутится, пользователь ждет.
> Решение: Анимации — приправа, не основное блюдо. `prefers-reduced-motion` обязательно.

**9. Нет единой визуальной системы.** Три шрифта, пять оттенков оранжевого.
> Решение: CSS-переменные, один-два шрифта, один акцентный цвет с вариациями.

**10. Форма с 10 полями.** Каждое лишнее поле — минус 10-15% конверсии.
> Решение: Для первого контакта: имя, способ связи, суть запроса. Все.

### Must-have элементы

| Элемент | Зачем |
|---|---|
| Hero с четким оффером | 3 секунды на захват внимания |
| Навигация с разделением тематик | Пользователь сразу выбирает свое |
| Портфолио/кейсы (digital) | Доказательство компетенций |
| Каталог объектов (недвижимость) | Конкретные предложения с ценами и фото |
| Социальное доказательство | Отзывы, цифры, логотипы |
| Множественные CTA | Форма, мессенджеры, телефон |
| О себе | Люди покупают у людей |
| Мобильная адаптация | Не nice-to-have, а обязательно |

### Топ-10 убийц конверсии

1. Когнитивный диссонанс от смешения тем
2. Медленная загрузка (каждая секунда = минус 7% конверсии)
3. Агрессивные попапы
4. Автовоспроизведение видео/аудио
5. Непонятная навигация с креативными названиями
6. «Цена по запросу» = «дорого и непрозрачно»
7. Неработающие элементы — мгновенная потеря доверия
8. Нечитаемый текст (оранжевый на темном — часто нечитаем в малых размерах!)
9. Нет обратной связи при отправке формы
10. Фейковые отзывы и стоковые фото

### Главный риск: совмещение двух тематик

**Как избежать провала:**

1. **Архитектура «хаб + спицы».** Главная — короткий хаб с двумя направлениями. Каждое — полноценный независимый лендинг.
2. **Связующий нарратив.** «Живу на Пхукете, создаю цифровые продукты и помогаю другим обосноваться здесь».
3. **Визуальное кодирование.** Оранжевый — разработка, бирюзовый — недвижимость.
4. **Раздельные точки входа.** Для рекламы — прямые URL на нужный раздел.

> **Итог UX-эксперта:** «Лучше сделать две простые, четкие страницы, чем одну красивую с размытым фокусом.»

## 7.2 Маркетолог, специалист по конверсии (12 лет опыта)

### Жесткая правда

> «Два бизнеса на одном лендинге — это маркетинговое самоубийство в 90% случаев. Совмещение возможно, но только через модель персонального бренда.»

### Офферы для цифровых услуг

**Варианты заголовков hero:**
1. «Сайты, приложения и ИИ-агенты, которые работают на ваш бизнес — пока вы спите»
2. «Ваш цифровой продукт — от идеи до работающего прототипа за 2 недели»
3. «Не ищите команду из 5 человек. Один специалист. Полный цикл. Работающий результат.»
4. «ИИ-агенты, которые заменят 3 сотрудников. Сайты, которые продают. Приложения, которые удерживают.»
5. «Цифровые решения от практика из Пхукета — для бизнеса без границ»

**Формула PAS:**
> **Problem:** Вы теряете клиентов каждый день, пока ваш сайт выглядит как из 2018 года.
> **Agitation:** Каждый месяц промедления — упущенные заявки и деньги.
> **Solution:** Я создаю цифровые продукты, которые работают с первого дня. Один специалист, полный цикл, без раздутых бюджетов.

### Офферы для недвижимости

**Варианты заголовков hero:**
1. «Недвижимость на Пхукете без посредников »
2. «Вилла на Пхукете с доходностью 8-12% годовых — подберу, проверю, проведу сделку»
3. «Пхукет: купить, арендовать, инвестировать»
4. «Не покупайте недвижимость на Пхукете, пока не прочитаете это»
5. «Ваша недвижимость на Пхукете — с цифровой прозрачностью каждого этапа»

### Как совместить и не размыть фокус

Единственный рабочий вариант — **персональный бренд + два направления:**

```
ГЛАВНАЯ (hero) — ТЫ как личный бренд
    |
    +-- DIGITAL (свой лендинг)
    +-- НЕДВИЖИМОСТЬ (свой лендинг)
```

**Hero главной:**
> « Создаю цифровые продукты. Помогаю найти недвижимость. Два направления — один принцип: все должно работать.»

Сразу после hero — два крупных CTA-блока:
- «Мне нужен цифровой продукт»
- «Мне интересна недвижимость»

### Триггеры доверия (когда мало кейсов)

**Для digital:**

| Триггер | Реализация |
|---------|-----------|
| Свой сайт как кейс | Сделай безупречно — он = кейс №1 |
| Процесс вместо результата | Секция «Как я работаю» с 4-5 шагами |

**Как создать доверие БЕЗ отзывов:**
1. Блог/канал с процессом работы в реальном времени (сильнее любого отзыва)
2. Гарантии с конкретикой
3. Бесплатная ценность вперед (аудит сайта, гайд по покупке)

## 7.3 Потенциальный клиент — владелец бизнеса (45 лет)

### Первые 5 секунд — что должно быть видно

- **Кто ты и чем конкретно полезен.** Не «цифровой мастер», а «Делаю сайты и приложения для бизнеса. Помогаю купить недвижимость на Пхукете.» Две строчки.
- **Лицо.** Реальное фото реального человека. Мне 45, я отдаю деньги людям, а не «брендам».
- **Цифры или имена.** «Сделал 40 проектов» или «Работал с такой-то компанией».
- **Где ты находишься.** Пхукет? Москва? Удаленно?

### Что вызывает подозрение

- Слишком много модных слов: «ИИ-агенты», «digital-экосистема», «инновационные решения» — когда все сразу, звучит пусто
- Нет ни одного реального кейса на первом экране
- Темная тема с оранжевым, которая выглядит как шаблон из Envato — должно быть **дорого и чисто**

### Реакция на совмещение digital + недвижимость

> «Первая мысль: он что, не может определиться, чем занимается?»

**Единственный рабочий вариант:** подать как историю образа жизни — «Живу на Пхукете. Делаю digital-продукты. Помогаю тем, кто хочет переехать или инвестировать.»

### Серьезный специалист vs очередной фрилансер

| Серьезный | Фрилансер |
|-----------|-----------|
| Показывает результаты клиентов | Показывает свои навыки |
| Говорит про деньги, сроки, процесс | Говорит про технологии и стек |
| 3-5 сильных кейсов | 20 мелких «проектиков» |
| Четкое позиционирование | «Делаю все» |
| Спокойный, уверенный тон | Продает себя агрессивно |

### 10 причин ОСТАТЬСЯ

1. Конкретный кейс с цифрами на первом экране
2. Видео-приветствие 30-60 секунд
3. Четкое разделение двух направлений
4. Портфолио: задача -> решение -> результат
5. Отзывы с фото и именами
6. Прозрачность по ценам (хотя бы вилки)
7. Описание процесса работы (4-5 шагов)
8. Реальные фото объектов с ценами
9. Быстрая загрузка и адаптация под мобильный
10. Личная история — почему Пхукет, как это связано

### 10 причин ЗАКРЫТЬ ВКЛАДКУ

1. Стена текста без структуры на первом экране
2. Ни одного реального проекта
3. Автоплей музыки или навязчивая анимация
4. «Делаю все: сайты, боты, ИИ, дизайн, маркетинг, SEO...» = ничего
5. Орфографические ошибки
6. Только форма, нет мессенджеров
7. Шаблонный дизайн (Bootstrap/типовой лендинг)
8. Попапы в первые 10 секунд
9. Нет цен и нет ориентира по бюджету
10. Долгая загрузка, битые картинки, неработающие кнопки

### Что заставит оставить заявку СЕЙЧАС

- Конкретный оффер с дедлайном: «Бесплатный аудит до конца недели»
- Простота: кнопка «Написать в Telegram», а не форма из 10 полей
- Кейс из моей отрасли
- Ощущение загруженности: «Беру 2-3 проекта в месяц»

### Два сайта vs один?

> «Честно? Два отдельных было бы лучше. Для digital — хочу видеть фокус. Секция про недвижимость заставляет думать: он на полставки программирует?»

> «Единственный рабочий вариант одного сайта — личный бренд: digital-услуги 70-80% сайта, недвижимость — дополнительный раздел.»

> «Темная тема с оранжевым — нормально, если сделано дорого. Но если выглядит как сайт геймера — аудитория 40-55 лет не воспримет всерьез. Для этой ЦА лучше сдержанная элегантность.»

## 7.4 СВОДНЫЕ РЕКОМЕНДАЦИИ (консенсус трех экспертов)

### Единогласное мнение: архитектура сайта

Все три эксперта сошлись: **два направления нельзя смешивать в одном потоке.** Рабочие варианты:

1. **Два отдельных лендинга** под одним доменом (`/digital` + `/phuket`)
2. **Хаб-страница** с личным брендом и двумя четкими входами
3. **Digital как основа** (70-80%), недвижимость как дополнительный раздел

### Связующая идея (нарратив)

> «Живу на Пхукете. Строю цифровые продукты для бизнеса. Помогаю тем, кто хочет найти свое место на острове.»

### ТОП-10 обязательных действий

1. **Четкое разделение направлений** — два CTA-блока на первом экране
2. **Сайт сам = кейс №1** — быстрый, красивый, безупречный
3. **Реальное фото и личное видео** (30-60 сек)
4. **Конкретика везде** — цифры, сроки, цены (хотя бы вилки)
5. **Mobile-first** — 70% трафика с телефонов
6. **Мессенджеры вместо форм** — WhatsApp/Telegram конвертят в 2-3 раза лучше
7. **3-4 CTA** по всей странице с разными формулировками
8. **Кейсы = задача -> решение -> результат** (не галерея скриншотов)
9. **Скорость загрузки < 3 сек** — оптимизация изображений, lazy loading
10. **Telegram-канал** — обязателен для длинного цикла (особенно недвижимость)

### Чего категорически НЕЛЬЗЯ делать

- Смешивать два оффера на одном экране
- Писать «делаю все» — нужно четкое позиционирование
- Ставить стоковые фото и фейковые отзывы
- Делать форму с 10 полями
- Игнорировать мобильную версию
- Использовать попапы и автоплей
- CTA-кнопка со словом «Отправить»
- Прятаться за логотипом/брендом — показать лицо

### Приоритеты на старте

1. Запустить digital-часть первой (быстрее приносит кейсы)
2. Недвижимость подключить как второе направление с контент-базой
3. Видео — секретное оружие доверия
4. Telegram-канал — воронка для длинного цикла продаж
